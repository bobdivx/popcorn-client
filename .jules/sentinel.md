## 2026-03-18 - Replacing insecure Math.random() with Web Crypto API / Node.js crypto
**Vulnerability:** Several utility functions generated sensitive identifiers (`generateId`, `generateInviteCode`, device IDs) or secure random bytes using insecure `Math.random()`. `Math.random()` generates deterministic, predictable output and is not cryptographically secure, making those identifiers easily guessable.
**Learning:** Some environments (like early versions of specific SSR handlers) might not support all Web Crypto API features or properly alias them to global objects. Fallbacks to insecure RNG functions can inadvertently bypass security properties entirely in modern deployment contexts unless an explicit fail-secure error is enforced.
**Prevention:** Always prioritize `globalThis.crypto.getRandomValues()` or `crypto.getRandomValues()`. Fallback to `require('crypto').randomBytes` in pure Node environments. *Never* fallback to `Math.random()` for generating UUIDs, invite codes, or sensitive tokens; if no secure PRNG is available, it is safer to throw a clear error to fail securely.
