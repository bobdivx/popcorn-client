## 2025-02-27 - Implicit Trust of Backend-Rendered HTML
**Vulnerability:** The `DescriptionPreview` component was implicitly trusting and rendering raw HTML injected by the backend (e.g., descriptions) via `dangerouslySetInnerHTML` without any prior validation or sanitization, introducing a critical Cross-Site Scripting (XSS) vulnerability.
**Learning:** In this codebase, data described as 'already rendered by the backend' cannot be implicitly trusted for safety on the frontend, especially if it may contain untrusted user input from an upstream source (e.g., tracker descriptions).
**Prevention:** Always use `isomorphic-dompurify` to sanitize backend-provided HTML content on the frontend before injecting it with `dangerouslySetInnerHTML`. The isomorphic variant guarantees safety across both Client-Side Rendering and Server-Side Rendering (SSR) environments, preventing hydration mismatches and SSR vulnerabilities.
## 2024-05-15 - Replace Math.random with cryptographically secure random number generation
**Vulnerability:** Weak random number generation using `Math.random()` was used for generating UUIDs, IDs, and other identifiers in various places across the application. `Math.random()` is predictable and not cryptographically secure, which could lead to ID collisions, predictable tokens, or security vulnerabilities if these IDs are used in security-sensitive contexts like tokens or session identifiers.
**Learning:** Even for non-critical identifiers like DOM element IDs or notification keys, using weak randomness can lead to subtle bugs due to collisions, especially in systems with high concurrency or fast generation rates. The project already had a Web Crypto API wrapper in `src/lib/utils/uuid.ts` but it was underutilized or lacked a standard UUID v4 implementation.
**Prevention:** Always use `globalThis.crypto.randomUUID()` or `globalThis.crypto.getRandomValues()` for generating random identifiers, tokens, or UUIDs. Avoid `Math.random()` entirely when any form of uniqueness or unpredictability is required. Ensure a centralized utility like `uuid.ts` provides accessible and standard compliant methods (e.g., `randomUUID()`) to discourage ad-hoc `Math.random()` implementations.
