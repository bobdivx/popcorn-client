## 2023-11-20 - [XSS] Missing HTML sanitization in DescriptionPreview
**Vulnerability:** The application was passing directly rendered or converted HTML to `dangerouslySetInnerHTML` in the `DescriptionPreview` component. This allowed for potential Cross-Site Scripting (XSS) if the HTML or BBCode source contained malicious scripts or payloads, leading to potential execution of arbitrary JavaScript in the user's session.
**Learning:** HTML generated from BBCode or passed directly from a backend/tracker is implicitly untrusted data on the client side. The implicit assumption that it is safe since it comes from the backend or the `bbcodeToHtml` utility is incorrect, as this raw content may contain arbitrary injected elements.
**Prevention:** Data passed to `dangerouslySetInnerHTML` must always be sanitized beforehand. `DOMPurify.sanitize` (or `isomorphic-dompurify` for server-side rendering/isomorphic environments) should be standard practice when rendering external HTML or parsed BBCode.
