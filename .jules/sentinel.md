## 2026-04-12 - Replaced Insecure PRNG in UUID Generation
**Vulnerability:** The application was using the cryptographically weak `Math.random()` function to generate internal UUIDs for indexers. This pattern is insecure and makes the resulting IDs highly predictable (CWE-338), potentially allowing an attacker to enumerate or collide identifiers.
**Learning:** In isomorphic environments, developers sometimes fall back to `Math.random()` when standard UUID libraries or Node `crypto` are not immediately available or to reduce bundle size. However, this codebase already has a centralized, cryptographically secure ID generator (`generateId` in `uuid.ts`) that prioritizes Web Crypto API and falls back gracefully.
**Prevention:** Always use the `generateId` utility from `src/lib/utils/uuid.ts` instead of manually constructing UUIDs using `Math.random()`. Ensure team members are aware of centralized utility functions for security-sensitive operations.
