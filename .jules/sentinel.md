## 2025-03-26 - [XSS] DescriptionPreview Component Sanitization Gap
**Vulnerability:** A Cross-Site Scripting (XSS) vulnerability was found in `src/components/upload/DescriptionPreview.tsx` where backend HTML was rendered directly via `dangerouslySetInnerHTML` with only partial placeholder removal (`replace(/\{\}/g, '')`), allowing malicious scripts to execute if the backend response was compromised or missing sanitization.
**Learning:** Even if data is explicitly marked as "already rendered by the backend" (`HTML dĂ©jĂ  rendu par le backend`), the frontend must independently sanitize raw HTML inputs before using `dangerouslySetInnerHTML`. Defense in depth dictates that the client-side should not implicitly trust API responses for HTML injection.
**Prevention:** Always use a robust isomorphic HTML sanitizer like `isomorphic-dompurify` (or a lightweight equivalent like `sanitize-html` if payload size is a concern) to sanitize any dynamic HTML content before binding it to `dangerouslySetInnerHTML`. Never rely on simple string replacement for security sanitization.
