Document Purpose and Scope

This UX Scenario Document describes in detail all user flows, screens, and test scenarios for the Popcorn application on LG webOS TV. It is intended for the LG QA team to perform functional and user-experience testing. Each scenario includes clear steps, expected results, and acceptance criteria.

1. Application Overview

Popcorn is a lightweight media client application that connects to a personal Popcorn media server. The application allows users to:

  • Browse and discover movies and TV shows from their personal media library
  • Search content across configured indexers
  • Manage and monitor torrent downloads remotely
  • Stream media content to their LG TV

Important: This application requires a connection to a Popcorn backend server. The app acts as a thin client - all media management runs on the user's personal server.

3. User Flow - Step by Step

3.1 Application Launch

Step 1: Launch the Popcorn application from the LG Content Store
Expected Result: The application loads and displays the main interface (hosted on Vercel)

3.2 Initial Configuration (First Launch Only)

Step 2: On first launch, the user is presented with a setup wizard
Actions:
  1. Select interface language (English or French)
  2. Enter server URL or use default (https://popcorn-vercel.vercel.app)
  3. Accept terms of service

3.3 Authentication

Step 3: Login screen is displayed
Actions:
  1. Enter email: lg-test@popcorn-app.com
  2. Enter password: LGTest2026!Secure
  3. Press "Sign in" button
Expected Result: User is authenticated and redirected to the home screen

3.4 Home Screen Navigation

Step 4: Home screen displays media content
Navigation:
  • Use Arrow keys (Up/Down/Left/Right) to navigate between items
  • Press OK/Enter to select an item
  • Press Back to return to previous screen
Available sections:
  • Continue Watching (resume previous content)
  • Popular Movies
  • Popular Series
  • Recent Additions

3.5 Search Function

Step 5: Access search from the navigation menu
Actions:
  1. Navigate to "Search" in the side menu
  2. Use on-screen keyboard to enter search query
  3. Results display movies and series matching the query

3.6 Content Playback

Step 6: Select and play content
Actions:
  1. Navigate to a movie or episode
  2. Press OK to view details
  3. Select "Play" to start streaming
Playback Controls:
  • Play/Pause: OK button or Play/Pause key
  • Rewind/Fast Forward: Left/Right arrows
  • Stop: Back button

3.7 Settings

Step 7: Access settings from the navigation menu
Available options:
  • Server Configuration
  • Account Settings
  • Language Preferences
  • Logout

8. Detailed Screen Descriptions

8.1 Splash / Loading Screen

Visual: Popcorn logo and/or loading indicator on screen.
Expected: Screen appears briefly while the app loads the web content from the server.
Duration: Typically 2–5 seconds depending on network.

8.2 Setup Wizard (First Launch)

Visual: Step-by-step wizard with language selector, server URL field, and terms of service checkbox.
Expected: User can select language (English/French), enter or keep default server URL, and accept terms.
Navigation: Arrow keys to move, OK to confirm each step.

8.3 Login Screen

Visual: Email and password fields, "Sign in" button.
Expected: User can enter credentials and submit. Error message appears if credentials are invalid.
Focus: Default focus on email field; Tab/Arrow keys move between fields.

8.4 Home Screen

Visual: Side navigation (Home, Search, Settings, etc.) and main content area with horizontal rows (Continue Watching, Popular Movies, Popular Series, Recent Additions).
Expected: Content loads from server. User can move focus with arrow keys and select items with OK.
Empty state: If no content, appropriate message is displayed.

8.5 Search Screen

Visual: On-screen keyboard and search input; results grid below.
Expected: User types query; results update as they type or after confirmation. Selecting a result opens detail view.

8.6 Content Detail Screen

Visual: Poster, title, description, "Play" button, possibly episode list for series.
Expected: User can start playback with Play or navigate to episodes.

8.7 Video Playback Screen

Visual: Full-screen video with optional on-screen controls (play/pause, seek, back).
Expected: Video plays; remote controls (OK, arrows, Back) work as described in section 4.

9. Error Handling and Edge Cases

9.1 Network Unavailable

Trigger: Device has no internet or server is unreachable.
Expected Result: Application displays a clear error message (e.g. "Unable to connect" or "Check your network") and optionally a retry option. No crash.
QA Check: Disconnect network, launch or refresh app; verify message and retry behavior.

9.2 Invalid Login Credentials

Trigger: User enters wrong email or password.
Expected Result: Error message such as "Invalid email or password" is shown; user remains on login screen and can correct input.
QA Check: Use wrong password with test account; verify message and focus.

9.3 Session Expired

Trigger: User session has expired on the server.
Expected Result: User is redirected to login screen or prompted to sign in again. No undefined behavior.
QA Check: If testable, expire session and perform an action; verify redirect.

9.4 Empty Content

Trigger: User has no movies/series or search returns no results.
Expected Result: Empty state message (e.g. "No content" or "No results") instead of blank screen or error.
QA Check: Use account with no library or search for "xyznonexistent"; verify message.

9.5 Back Button Behavior

Trigger: User presses Back repeatedly from various screens.
Expected Result: Navigation goes back (detail → list → home) and eventually exits app or shows exit confirmation, per webOS guidelines. No crash.
QA Check: From playback, detail, search, and home; verify consistent back behavior.

10. Notes for QA Team

📋 Important Notes

  1. The application loads content from an external server (Vercel). Please ensure the test device has internet connectivity.
  2. The test account provided above has been pre-configured with sample content.
  3. All navigation is optimized for TV remote control - no mouse or touch input required.
  4. The application supports both English and French languages.
  5. If you encounter any connection issues, please verify the server URL is set to: https://popcorn-vercel.vercel.app
  6. For defect resolution, providing a short proof video or screenshots of the issue is helpful.
